You can use SageMath, a mathematics software system built on top of Python 3 to work with prime curves. Here's an example in which we use to define the curve with as in the example above:

# Define the prime curve with p = 23, a = 1, b = 1
E = EllipticCurve(GF(23), [1, 1])

# Get a (random) generator for the field. Since the field 
# is cyclic, this will only return one value as per 
# SageMath's documentation.
generators = E.gens()
P = generators[0]
print(f"Generator point: P = {P.xy()}\n")

# List all the points, *excluding* the point at infinity
# In SageMath, points are expressed as 3-tuples (x : y : infinity);
# if P[2] is 1, the point represents the point at infinity.
points = [P.xy() for P in E.points() if P[2]]
print(f"Points: {points}\n")

# How many points *including the point at infinity* are there?
print(f"Cardinality of the set: {E.cardinality()}\n")

# Let's double the generator point just to see where it goes
print(f"2P = {(2 * P).xy()}\n")

With all of this background defined, we can now talk about how to use these curves towards cryptography.

Remember that factoring the coprime numbers of an RSA key is really hard, and is where its security comes from; the only option to undo modular exponentiation (as is done when encrypting messages) is to either have the private key or brute force every possible value that could be the private key.

Addition in ECC is similar to modular multiplication in RSA, while the act of adding things many times in ECC is similar to modular exponentiation in RSA. That is to say, the "hard" problems of ECC correspond to the difficulty of factoring the product of two large primes or taking the discrete logarithm.
